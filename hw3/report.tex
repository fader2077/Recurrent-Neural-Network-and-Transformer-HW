\begin{abstract}
This report presents the official Homework 3 pipeline using a 50-question random sample from the specified Kaggle \texttt{train.csv} file with \texttt{random\_state=42}. The system compares two chunking strategies, builds two FAISS indices with \texttt{BAAI/bge-m3}, applies two-stage retrieval with dense search followed by \texttt{cross-encoder/ms-marco-MiniLM-L-6-v2}, and generates final answers with local Ollama \texttt{llama3}. On the official evaluation set, the final generation pipeline reached \textbf{80.00\% accuracy (40/50)}. A previous science-only / DeepSeek / ensemble branch is retained only as an optional appendix and is not used as the formal HW3 score.
\end{abstract}

\section{Introduction}

The goal of Homework 3 is to build a retrieval-augmented generation pipeline for science question answering. The official submission in this report follows the required structure:

\begin{itemize}
    \item Part 1: compare two chunking strategies and build two vector indices.
    \item Part 2: implement dense retrieval plus cross-encoder re-ranking and compare hit rate.
    \item Part 3: use Ollama Llama-3 to answer multiple-choice questions from retrieved evidence.
    \item Latency analysis: measure vector search, re-ranking, and generation time on the same official pipeline.
\end{itemize}

\section{Part 1: Indexing Pipeline}

\subsection{Chunking Strategy A}

Strategy A uses fixed-size recursive chunking with \texttt{chunk\_size=500} and \texttt{overlap=50}. This is the smaller-granularity baseline index.

\subsection{Chunking Strategy B}

Strategy B uses larger recursive chunks with \texttt{chunk\_size=1000} and \texttt{overlap=200}. This keeps more local context inside each chunk.

\subsection{Vector Index Construction}

Both indices use the same embedding model and GPU execution:

\begin{itemize}
    \item Embedding model: \texttt{BAAI/bge-m3}
    \item Vector store: FAISS
    \item Device: CUDA GPU
\end{itemize}

Index A stores Strategy A chunks, and Index B stores Strategy B chunks.

Strategy A creates many shorter chunks, while Strategy B produces fewer but longer chunks. In this rerun, smaller chunks more frequently cut sentence boundaries and split complete answer spans across chunk edges, while Strategy B preserved longer contiguous evidence and gave more reliable complete-answer capture.
The larger Strategy B chunks reduce fragmentation of multi-sentence explanations, which is important for science questions where the answer often depends on a definition plus a consequence. The tradeoff is that longer chunks may contain more irrelevant surrounding text, so re-ranking becomes more important.

\section{Part 2: Retrieval and Re-ranking}

\subsection{Dense Retrieval}

Dense retrieval is the high-recall first stage. It encodes the question into the same embedding space as the indexed chunks and uses vector similarity to quickly return top-\texttt{k} candidates. This stage is computationally efficient and scales well, so it is suitable for broad candidate recall. The tradeoff is that embedding similarity is approximate: dense top-1 can be semantically adjacent or lexically similar, but still not the most answer-bearing chunk.

\subsection{Cross-Encoder Re-ranking}

Cross-encoder re-ranking is the high-precision second stage. For each query--chunk pair from the dense candidate set, the model jointly encodes both texts and scores relevance with token-level interactions. This usually gives finer discrimination than embedding-only retrieval, especially for ambiguous terms and dense top-1 ordering mistakes. It is slower than vector search, but the cost stays controlled because re-ranking is applied only to the dense top-\texttt{k} candidates rather than the full index.

\subsection{Hit Rate Comparison}

Hit rate is reported as a retrieval proxy metric based on keyword overlap between the retrieved chunks and the correct answer text. The assignment requires this comparison, so it is kept explicitly as a proxy retrieval metric rather than a perfect relevance label.
A question is counted as a hit if at least one token from the correct answer text (after stopword removal) appears in at least one of the final top-3 chunks. This is a proxy metric rather than a human relevance label.

\subsection{Two Qualitative Re-ranking Examples}

The following two cases were selected from the official random-50 evaluation set using the final Index A retrieval branch. They show cases where vector search ranked a weak or irrelevant chunk first, while cross-encoder re-ranking promoted a more answer-relevant chunk to rank 1.
These examples were selected by scanning the official random-50 set for cases where the dense top-1 chunk was off-topic and the cross-encoder promoted a higher-scoring, more answer-relevant chunk to rank 1.

\textbf{Case 1: Planetary-system concept promotion}

\begin{itemize}
    \item Source: official random-50
    \item Question: What is a planetary system?
    \item Correct answer: \texttt{C. Any set of gravitationally bound non-stellar objects in or out of orbit around a star or star system.}
    \item Vector Search top-1:
    \item snippet: \texttt{A planet is a large object such as Venus or Earth that orbits a star ...}
    \item score: \texttt{2.6344}
    \item why it is weak / irrelevant: this chunk defines a single planet and does not directly describe the broader planetary-system concept in the answer.
    \item Re-ranked top-1:
    \item snippet: \texttt{Planetary astronomy is the study of planets ... and other small objects that orbit stars ...}
    \item original vector rank: \texttt{2}
    \item score: \texttt{4.2828}
    \item evidence phrase: \texttt{objects that orbit stars}
    \item why it is more answer-relevant / supports the correct answer: it directly addresses the system-level definition around objects orbiting stars.
    \item Explanation: cross-encoder re-ranking promoted a broader, more answer-bearing chunk from rank 2 to rank 1 with a higher score.
\end{itemize}

\textbf{Case 2: Molecular-cloud formation mechanism promotion}

\begin{itemize}
    \item Source: official random-50
    \item Question: What is the reason for the formation of stars exclusively within molecular clouds?
    \item Correct answer: \texttt{B. Low temperature and high density allow gravity to exceed outward pressure and trigger collapse.}
    \item Vector Search top-1:
    \item snippet: \texttt{Many galaxies continue to form new generations of stars ...}
    \item score: \texttt{-1.4520}
    \item why it is weak / irrelevant: this is broad star-formation background and misses the molecular-cloud collapse mechanism.
    \item Re-ranked top-1:
    \item snippet: \texttt{The Sun formed by gravity in a large molecular cloud ...}
    \item original vector rank: \texttt{25}
    \item score: \texttt{-0.1704}
    \item evidence phrase: \texttt{formed by gravity in a large molecular cloud}
    \item why it is more answer-relevant / supports the correct answer: it explicitly links star formation to molecular clouds and gravitational collapse context.
    \item Explanation: cross-encoder re-ranking promoted a mechanism-bearing chunk from rank 25 to rank 1 with a clear score gain.
\end{itemize}

These examples are qualitative evidence-ordering demonstrations and do not change the official random-50 accuracy computation.
\section{Part 3: Ollama Generation}

\subsection{Llama-3 Integration}

The official generator is local Ollama \texttt{llama3}.

\subsection{Anti-hallucination Prompt}

The system prompt enforces two constraints:

\begin{itemize}
    \item use only the retrieved context,
    \item output exactly one letter from \texttt{A/B/C/D/E}.
\end{itemize}

For open-ended RAG usage, the same policy maps insufficient evidence to ``I do not know'', while this HW3 MCQ run enforces single-letter output.

The generator receives the reranked top-3 chunks together with the question and answer choices.

Prompt skeleton:
\begin{verbatim}
Context:
[retrieved chunks]

Question:
...

Choices:
A ...
B ...
C ...
D ...
E ...

Instruction:
Use only the context. Output exactly one letter.
\end{verbatim}

\section{Conclusion}

The final official HW3 pipeline uses a random 50-question sample from the specified Kaggle \texttt{train.csv} path, two chunking strategies, FAISS retrieval, cross-encoder re-ranking, and Ollama \texttt{llama3} generation on GPU. Strategy B gives the strongest retrieval proxy hit rate, but the end-to-end ablation shows the best final QA accuracy comes from Index A with re-ranking, so that setting is selected for the formal result. The appendix shows that a science-only DeepSeek/ensemble branch can reach a higher ceiling, but the formal HW3 score reported here remains 80.00\%.
