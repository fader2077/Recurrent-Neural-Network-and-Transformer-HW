Part 2: The Retrieval & Re-ranking System (40%) 
Retrieval is rarely perfect on the first try. You will implement a two-stage retrieval process. 
1. Stage 1: Dense Retrieval (Vector Search) 
o Query your Vector DB to retrieve the top-K (e.g., k=20) candidates. 
2. Stage 2: Re-ranking (The "Secret Sauce") 
o Implement a Cross-Encoder (e.g., cross-encoder/ms-marco-MiniLM-L-6-v2) to 
score the relevance between the User Query and the 20 retrieved candidates. 
o Sort the candidates by the Cross-Encoder score and select the top-N (e.g., k=3) 
final context chunks. 
o Why? Vector search is fast but approximate; Cross-Encoders are slow but accurate.  
3. Experiment: Compare the retrieval quality (Hit Rate) of "Vector Search Only" vs. "Vector

This is where you utilize the RTX 4090 to boost accuracy.

from sentence_transformers import CrossEncoder 
# Load Cross-Encoder on GPU 
reranker = CrossEncoder('cross-encoder/ms-marco-MiniLM-L-6-v2', 
device='cuda') 
def advanced_retrieve(query, vector_db, initial_k=20, final_k=3): 
# Stage 1: Fast Retrieval 
candidates = vector_db.similarity_search(query, k=initial_k) 
# Stage 2: Re-ranking 
# Prepare pairs for the Cross-Encoder: [[Query, Doc1], [Query, 
Doc2], ...] 
candidate_texts = [doc.page_content for doc in candidates] 
pairs = [[query, doc_text] for doc_text in candidate_texts] 
# Get scores 
scores = reranker.predict(pairs) 
# Sort by score (descending) 
scored_candidates = sorted(zip(scores, candidates), key=lambda x: 
x[0], reverse=True) 
# Return top-N highly relevant docs 
final_docs = [doc for score, doc in scored_candidates[:final_k]] 
return final_docs
